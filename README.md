# MBTI 性格测试小程序原型

一个面向移动端体验的 MBTI 测试项目，覆盖首页引导、逐题答题、状态管理、计分逻辑、结果解析和图表可视化。项目重点展示我如何使用 Codex 从需求澄清推进到可运行原型、测试脚本和作品集文档。

## 技术栈

- HTML5 / CSS3 / JavaScript
- Chart.js：结果页维度得分可视化
- Node.js：运行计分逻辑自动化测试
- Codex：辅助需求拆解、数据结构设计、前端实现、状态管理和测试方案生成

## 本地运行

直接打开 `index.html` 即可体验。

也可以使用本地静态服务：

```bash
python -m http.server 5173
```

访问：

```text
http://localhost:5173
```

运行计分测试：

```bash
node mbti-scoring.test.js
```

## 项目结构

```text
.
├── index.html              # 页面结构：首页、答题页、结果页
├── questions-data.js       # 82 道题库，含 80 道核心题与 2 道一致性题
├── styles.css              # 移动端布局、视觉样式、卡片切换动画
├── app.js                  # 题库、状态管理、答题导航、计分、图表渲染
├── mbti-scoring.test.js    # 纯 JavaScript 计分测试脚本
└── PORTFOLIO_NOTES.md      # 作品集文案与面试亮点
```

## 核心功能

- 首页介绍与开始测试入口
- 逐题答题、选项切换、上一题/下一题导航
- 使用 `state` 统一管理当前题号、答题记录、完成状态和结果
- 根据 E/I、S/N、T/F、J/P 四个维度计算 16 型人格结果
- 结果页展示人格类型、描述、优势和职业建议
- Chart.js 展示维度得分分布
- 自动化测试验证 ENFJ、ISTP、INFP 三组典型答题路径

## Showcase 文案

移动端 MBTI 测试原型，包含逐题答题、状态管理、16 型结果解析与维度图表可视化，展示从需求到测试的完整前端实现。



1. 我用 Codex 先做需求澄清，把 MBTI 测试拆成页面流程、题目结构、答题记录、计分规则和结果映射，避免一开始就过度设计。
2. 我用 Codex 辅助实现集中式状态管理，用一个 `state` 对象驱动当前题号、答题记录、完成状态、重测和结果跳转，保证交互逻辑一致。
3. 我用 Codex 生成测试方案和纯 JavaScript 自动化测试脚本，通过 ENFJ、ISTP、INFP 三组模拟路径验证计分函数，提升结果可信度。
